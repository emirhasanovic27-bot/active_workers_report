How it works. Each Custom Export is transaction-level (one row per day worked). The app de-duplicates to one row per active worker (by Username), keeping the most recent / highest-level entry. The Location column is never read from the export — it is looked up in an invisible internal database built from the previous monthly reports. If a username isn't in the database, Location is left blank. Google takes two exports — GTNP and EMEA — which merge into the one Google tab (GTNP first, then EMEA); upload only GTNP if there's no EMEA. Hitachi is a single export that is automatically split into three tabs by team — Brand Refresh → Hitachi BR MS workers, Marketing & Creative Shared Services → Hitachi DC MS workers, Digital Communications & Marketing → Hitachi DM MS workers.
